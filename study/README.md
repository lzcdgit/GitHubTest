git add & git commit
看到这里估计很多人会有疑问，我想要提交直接进行 commit 不就行了么，为什么先要再 add
一次呢？首先 git add 是先把改动添加到一个「暂存区」，你可以理解成是一个缓存区域，临
时保存你的改动，而 git commit 才是最后真正的提交。这样做的好处就是防止误提交，当然
也有办法把这两步合并成一步，不过后面再介绍，建议新手先按部就班的一步步来。
